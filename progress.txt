--- Pipeline Summary ---
data_collection.py → feature_engineering.py → clustering.py → classification.py → anomaly_detection.py → dashboard.py

Key design decisions:
  - KMeans clusters used as classification target (cluster-then-predict pattern)
  - log1p + RobustScaler preprocessing consistent across all modules
  - Anomaly detection uses auto threshold (not arbitrary contamination %)
  - Two-layer anomalies: global (IF) + intra-cluster (Mahalanobis)
  - SHAP per-cluster explanations for interpretability
  - Model bundle includes all preprocessing params for real-time inference
